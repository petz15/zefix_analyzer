{# ── Claude Classification ──────────────────────────────────────────────── #}
Claude Classification

Classify companies using Claude Haiku. Stores a claude_score (0-100) and claude_category per company. Requires an Anthropic API key (set in Claude API Options above). Per-request: ~$0.37 / 1k companies (20 company/msg). With 20 companies/msg + prompt caching: ~$0.15 / 1k. Batch API adds another ~50% discount. Number of companies per msg has a large impact on costs. Bulk runs can take upto 24h. Using the fixed category list and auto-filtering by target keywords can improve consistency and reduce costs by skipping irrelevant companies.

{# ── TF-IDF Clustering ──────────────────────────────────────────────────── #}
TF-IDF Clustering

Cluster companies by purpose text using spaCy lemmatization + TF-IDF + TruncatedSVD + HDBSCAN. Stores a comma-separated label (top-N terms) in tfidf_cluster. Noise/outliers are set to null. Leave filters empty to process all companies.

Advanced clustering parameters

Analyze which terms appear across many cluster labels — use after clustering to find new stopword candidates. Output saved to cluster_analysis.txt.

Recompute purpose_keywords without re-clustering. Runs lemmatization + TF-IDF only — tfidf_cluster is left unchanged. Use after tuning stopwords or before a keyword-mode cluster run.
